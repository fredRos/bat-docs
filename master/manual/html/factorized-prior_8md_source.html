1 Defining a factorized prior {#cha-factorized-prior}
2 ================
3 
4 [TOC]
5 
6 If a model does not overload `BCModel::LogAPrioriProbability`,
7 then its prior is the product of the individual priors for each of its
8 parameters. We call these the factorized priors. To set a parameter's
9 prior call
10 @code{.cpp}
11 BCParameter::SetPrior(BCPrior* const prior);
12 @endcode
13 You can call factorized priors within your overloaded
14 `LogAPrioriProbability` by querying the log of the prior of a
15 particular parameter with
16 @code{.cpp}
17 BCParameter::GetLogPrior(double x)
18 @endcode
19 The prior you set for a parameter need only inherit from
20 `BCPrior`. BAT has several built in prior classes, such as
21 `BCConstantPrior` and `BCGaussianPrior`. You can implement
22 new factorized priors by inheriting from the `BCPrior` class. To
23 illustrate how to do this, we will work through the construction of
24 the Gaussian prior:
25 @code{.cpp}
26 class BCGaussianPrior : public BCPrior
27 @endcode
28 `BCPrior` is a pure-virtual class with three methods that must be overloaded:
29 @code{.cpp}
30 double GetLogPrior(double x);
31 BCPrior* clone() const;
32 bool IsValid() const;
33 @endcode
34 The first one contains the meat of our new prior:
35 @code{.cpp}
36 double GetLogPrior(double x)
37 {
38  return -0.5 * (x - fMean) * (x - fMean) / fSigma / fSigma - log(fSigma) - 0.5 * log(2 * M_PI);
39 }
40 @endcode
41 This is, naturally, the log of a Guassian prior. The second one should
42 simply return a copy of the prior:
43 @code{.cpp}
44 BCPrior* clone() const
45 {
46  return new BCGaussianPrior(*this);
47 }
48 @endcode
49 The last function is required for checking that everything that is
50 needed by the prior is properly set. You'll notice that our example
51 calls on two member variables in its `GetLogPrior`: `fMean`
52 and `fSigma`. So we must check whether they have been properly set:
53 @code{.cpp}
54 bool IsValid() const
55 {
56  return std::isfinite(fMean) and std::isfinite(fSigma) and fSigma > 0;
57 }
58 @endcode
59 This checks that the mean and standard deviation are both finite and
60 that the standard deviation is positive semi-definite, as it need be.
61 
62 Naturally to be of use, we also create a constructor that allows us to
63 set the mean and standard deviation at creation; and getters and
64 setters that allow us to access them. (See the source code of
65 `BCGaussianPrior` for their implementation.)
66 
67 This is all that is required to create a new prior. `BCPrior` has
68 several methods for getting properties of the prior: the mode; the
69 integral over a range; the \f$n\f$'th raw, central, and standardized
70 moments; the mean; the variance; the standard deviation; the skewness;
71 and the kurtosis. These functions make their own internal calculations
72 and need not be overloaded. If you wish to speed them up, or provide
73 exact results, you may overload them. But take care that you overload
74 them to give back proper results! For example, the mode of the
75 Gaussian distribution is not simply `fMean`, since this presumes
76 the range over which we query contains `fMean`. The more proper
77 implementation is
78 @code{.cpp}
79 double GetMode(double xmin, double xmax)
80 {
81  if (fMean < xmin)
82  return xmin;
83  if (fMean > xmax)
84  return xmax;
85  return fMean;
86 }
87 @endcode
