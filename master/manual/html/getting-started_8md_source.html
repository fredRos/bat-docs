50 @section sec-basics-define-a-model Defining a model
51 
52 To define a valid BAT model we must make three additions to the empty
53 model that `bat-project` has created:
54 1. we must add parameters to our
55 model;
56 2. we must implement a log-likelihood function;
57 3. and we must implement or state our priors.
58 
59 We will start with a simple model that fits a normal distribution to
60 data, and so has three parameters: the mode (\f$\mu\f$) and standard
61 deviation (\f$\sigma\f$) of our distribution, and a scaling factor
62 ("height"). We will start with flat priors for all.
63 
64 How you store and access your data is entirely up to you. For this
65 example, we are going to fit to a binned data set that we store as a
66 ROOT histogram in a private member of our model class. In the header,
67 add to the class
68 
69 @code{.cpp}
70 
71 #include <TH1D.h>
72 
73 ...
74 
75 class MyMod : public BCModel
76 {
77 
78  ...
79 
80 private:
81  TH1D fDataHistogram
82 
83 };
84 @endcode
85 
86 And in the source file, we initialize our `fDataHistogram` in the
87 constructor; let's also fill it with some random data, which ROOT can
88 do for us using a TF1:
89 
90 @code{.cpp}
91 #include "MyMod.h"
92 #include <TF1.h>
93 
94 ...
95 
96 // ---------------------------------------------------------
97 MyMod::MyMod(const std::string& name)
98  : BCModel(name),
99  fDataHistogram("data", ";mass [GeV];count", 100, 5.0, 5.6)
100 {
101  // create function to fill data according to
102  TF1 data_func("data_func", "exp(-0.5*((x - 5.27926) / 0.04)^2)", 5.0, 5.6);
103 
104  // fill data histogram randomly from data_func 1,000 times
105  fDataHistogram.FillRandom("data_func", 1e3);
106 }
107 @endcode

109 @subsection subsec-basics-add-parameters Adding parameters
110 
111 To add a parameter to a model, call its member function `BCModel::AddParameter`:
112 @code{.cpp}
113 bool AddParameter(const std::string& name, double min, double max, const std::string& latexname = "", const std::string& unitstring = "")
114 @endcode
115 
116 You must indicate a parameter's name and its allowed range, via `min`
117 and `max`. Each parameter must be added with a unique name. BAT will
118 also create a "safe version" of the name which removes all non
119 alpha-numeric characters except for the underscore, which is needed
120 for naming of internal storage objects. Each parameter name should
121 also convert to a unique safe name; BAT will complain if this is not
122 the case (and `AddParameter` will return `false`).
123 
124 Optionally, you may add a display name for the parameter and a unit
125 string, both of which are used when BAT creates plots.
126 
127 The most logical place to add parameters to a model is in its
128 constructor. Let us now edit `MyMod.cxx` to add our mode, standard
129 deviation, and height parameters inside the constructor:
130 
131 @code{.cpp}
132 MyMod::MyMod(const std::string& name)
133  : BCModel(name),
134  fDataHistogram("data", ";mass [GeV];count", 100, 5.0, 5.6)
135 {
136  ...
137 
138  AddParameter("mu", 5.27, 5.29, "#mu", "[GeV]");
139  AddParameter("sigma", 25e-3, 45e-3, "#sigma", "[GeV]");
140  AddParameter("height", 0, 10, "", "[events]");
141 }
142 @endcode
143 
144 I have chosen the ranges because I have prior information about the
145 data: the mode of my distribution will be between 5.27 and 5.29; the
146 standard deviation will be between 0.025 and 0.045; and the height
147 will be between 0 and 10. When we provide a prior for a parameter,
148 \f$p_0(\lambda)\f$, the prior BAT uses is
149 
150 \f{equation}{
151  P_0(\lambda) = \begin{cases} p_0(\lambda),& \text{if }
152  \lambda\in[\lambda_{\text{min}}, \lambda_{\text{max}}],\\ 0,&
153  \text{otherwise.}
154  \end{cases}
155 \f}
156 
157 So keep mind that the ranges your provide for parameters become part
158 of the prior: BAT will not explore parameter space outside of the
159 range limits you provide.
160 
161 If you have written the code correctly for adding parameters to your
162 model, your code should compile. BAT will again, though, issue an
163 error if you try to run `runMyTut`, since we are still missing priors.
164 
165 @subsection subsec-basics-setting-priors Setting prior distributions
166 
167 There are two ways we may set the prior, \f$P_0(\vec\lambda)\f$ for a parameter
168 point: We can override the function that returns the log _a priori_ probability
169 for a model (`BCModel::LogAPrioriProbability`) and code anything we can dream of
170 in C++:
171 
172 @code{.cpp}
173 double MyMod::LogAPrioriProbability(const std::vector<double>& pars)
174 {
175  ...
176 }
177 @endcode
178 
179 In this case, you have to make sure the prior is properly normalized if you wish to compare different models as BAT will not do this for you.
180 
181 Or we can set individual priors for each parameter, and BAT will multiply them
182 together for us including the proper normalization. If each parameter has a
183 prior that factorizes from all the others, then this is the much better option.
184 In this case, we **must not** override `LogAPrioriProbability`. Instead we tell
185 BAT what the factorized prior is for each parameter, by adding a `BCPrior`
186 object to each parameter after we create it:
187 
188 @code{.cpp}
189 #include <BAT/BCGaussianPrior.h>
190 
191 ...
192 
193 MyMod::MyMod(const std::string& name)
194  : BCModel(name),
195  fDataHistogram("data", ";mass [GeV];count", 100, 5.0, 5.6)
196 {
197  ...
198 
199  // add parameters for Gaussian distribution
200  AddParameter("mu", 5.27, 5.29, "#mu", "[GeV]");
201  GetParameters().Back().SetPrior(new BCGaussianPrior(5.28, 2e-3));
202 
203  AddParameter("sigma", 25e-3, 45e-3, "#sigma", "[GeV]");
204  GetParameters().Back().SetPrior(new BCGaussianPrior(35e-3, 3e-3));
205 
206  AddParameter("height", 0, 10, "", "[events]");
207  GetParameters().Back().SetPriorConstant();
208 }
209 @endcode
210 
211 We could create a `BCConstantPrior` object for `height` just as we
212 created a `BCGaussianPrior` object for `mu` and `sigma`, but BAT has a
213 convenient function that creates one for us.
214 
215 We can now compile our code and run it. The results will be
216 meaningless, though, since we have yet to define our likelihood.

218 @subsection subsec-basics-loglikelihood Defining a likelihood
219 
220 The heart of our model is the likelihood function---more specifically,
221 since BAT works with the natural logarithm of functions, our
222 log-likelihood function.
223 
224 Given a histogrammed data set containing numbers of events in each
225 bin, our statistical model is the product of Poisson probabilities for
226 each bin---the probability of the events observed in the bin given the
227 expectation from our model function:
228 
229 \f{equation}{
230  \mathcal{L}(\vec\lambda) \equiv \prod_{i} \text{Poisson}(n_i|\nu_i),
231 \f}
232 
233 where \f$n_i\f$ is the number of events in bin \f$i\f$ and \f$\nu_i\f$
234 is the expected number of events given our model, which we will take
235 as the value of our model function at the center of the bin,
236 \f$m_i\f$,
237 
238 \f{equation}{
239  \nu_i \equiv f(m_i) = \frac{1}{\sqrt{2\pi}\sigma}\exp{\left(-\frac{(m_i - \mu)^2}{2\sigma^2}\right)}.
240 \f}
241 
242 Working with the log-likelihood, this transforms into a sum:
243 
244 \f{equation}{
245  \log\mathcal{L}(\vec\lambda) \equiv \sum_{i} \log\text{Poisson}(n_i|\nu_i).
246 \f}
247 
248 BAT conveniently has a function to calculate the logarithm of the
249 Poisson distribution (with observed \f$x\f$ and expected
250 \f$\lambda\f$) for you:
251 
252 @code{.cpp}
253 double BCMath::LogPoisson(double x, double lambda)
254 @endcode
255 
256 Let us code this into our log-likelihood function:
257 
258 @code{.cpp}
259 
260 #include <BAT/BCMath.h>
261 
262 #include <TMath.h>
263 
264 ...
265 
266 // ---------------------------------------------------------
267 double MyMod::LogLikelihood(const std::vector<double>& pars)
268 {
269  // store our log-likelihood as we loop through bins
270  double LL = 0.;
271 
272  // loop over bins of our data
273  for (int i = 1; i <= fDataHistogram.GetNbinsX(); ++i) {
274 
275  // retrieve observed number of events
276  double x = fDataHistogram.GetBinContent(i);
277 
278  // retrieve bin center
279  double m = fDataHistogram.GetBinCenter(i);
280 
281  // calculate expected number of events, using ROOT Gaus function
282  double nu = TMath::Gaus(m, pars[0], pars[1], true);
283 
284  // add to log-likelihood sum
285  LL += BCMath::LogPoisson(x, nu);
286 
287  }
288 
289  // return log-likelihood
290  return LL;
291 }
292 @endcode

349 @section sec-basics-observables Adding an observable
350 
351 We can store the posterior distribution of any function of the
352 parameters of our model using BAT's `BCObservable`'s. Let us suppose
353 we want to know the posterior distribution for the total number of
354 events our model predicts---the signal yield; and we want to know the
355 standard deviation's relation to the mean: \f$\sigma/\mu\f$. In our
356 constructor, we add the observables in the same way we added our
357 parameters:
358 
359 @code{.cpp}
360 // ---------------------------------------------------------
361 MyMod::MyMod(const std::string& name)
362  : BCModel(name),
363  fDataHistogram("data", "mass [GeV];count", 100, 5.0, 5.6)
364 {
365  ...
366 
367  AddObservable("SignalYield", 900, 1100, "Y_{S}", "[events]");
368  AddObservable("Resolution",
369  100. * GetParameter("sigma").GetLowerLimit() / GetParameter("mu").GetUpperLimit(),
370  100. * GetParameter("sigma").GetUpperLimit() / GetParameter("mu").GetLowerLimit(),
371  "#sigma / #mu", "[%]");
372 }
373 @endcode
374 
375 Note that an observable does not need a prior since it is not a
376 parameter; but it does need a range for setting the marginalized
377 histogram's limits. Since we know already that the answer will be 1000
378 events with an uncertainty of \f$\sqrt{1000}\f$, we set the range for
379 the yield to \f$[900, 1100]\f$.
380 
381 And we need to calculate this observable in our
382 `CalculateObservables(...)` member function. Uncomment it in the
383 header file `MyMod.h`:
384 
385 @code{.cpp}
386  void CalculateObservables(const std::vector<double> & pars);
387 @endcode
388 
389 and implement it in the source:
390 @code{.cpp}
391 // ---------------------------------------------------------
392 void MyMod::CalculateObservables(const std::vector<double>& pars)
393 {
394  // store total of number events expected
395  double nu = 0;
396 
397  // loop over bins of our data
398  for (int i = 1; i <= fDataHistogram.GetNbinsX(); ++i)
399  // calculate expected number of events in that bin
400  // and add to total expectation
401  nu += pars[2] * TMath::Gaus(fDataHistogram.GetBinCenter(i), pars[0], pars[1], true);
402 
403  // store in the observable
404  GetObservable(0) = nu;
405 
406  // Store sigma as percentage of mu:
407  GetObservable(1) = 100. * pars[1] / pars[0];
408 }
409 @endcode
410

412 distributions and text output for the observables:

448 m.PrintAllMarginalized(m.GetSafeName() + "_plots.pdf", 2, 2);
