1 Sharing / Loading samples {#cha-sharing-samples}
2 =================
3 
4 [TOC]
5 
6 You can output the samples of BAT's Markov chains in the form of a
7 ROOT `TTree` saved in a `TFile`. You can read these samples
8 back into BAT using `BCEmptyModel`:
9 @code{.cpp}
10 BCEmptyModel m("MyModel_mcmc.root");
11 m.Remarginalize();
12 @endcode
13 where `MyModel\_mcmc.root` is a `.root` file produced by a BAT
14 analysis---that is, it contains two `TTree`'s, `X\_mcmc.root`
15 and `X\_pars.root`, where X is a prefix common to both trees.
16 BAT
17 will search the file for two such trees matching the structures
18 expected from BAT output.
19 Alternatively, the constructor can be
20 called with the prefix specified explicitly:
21 @code{.cpp}
22  BCEmptyModel::BCEmptyModel(const std::string& filename, const std::string& name, bool loadObservables).
23 @endcode
24 The last argument tells switches on or off the loading of
25 `BCObservable`'s stored in the `TTree`.
26 
27 You can also use an instance of your own model instead of `BCEmptyModel` if you provide a
28 constructor that calls the
29 @code{.cpp}
30 BCModel::BCModel(const std::string\&
31 filename, const std::string\& name, bool loadObservables)
32 @endcode
33 constructor.

36 
37 Note that instead of calling `MarginalizeAll()`, a reloaded BAT
38 analysis is marginalized by calling `Remarginalize()`.
39 
40 All subsequent drawing or accessing of `BCH1D` and `BCH2D`
41 objects is the same as if the analysis had been run directly rather
42 than reloaded. For most summary reports of BAT, the actual model
43 itself is not needed---only the samples. So you can share your results
44 by sharing your ROOT output files without sharing your model
45 implementation.
46 
47 Likewise, when outputting to a ROOT file, BAT (by default) autosaves
48 the Markov chains to the ouput file at regular intervals. You can use
49 the steps outlined above to check the state of an ongoing analysis
50 using `BCEmptyModel`.
