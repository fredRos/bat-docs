8 the multi-template fitter (mtf) is a tool which allows to fit several
9 template histograms to a data histogram. the content of the bins in the
10 templates are assumed to fluctuate independently according to poisson
11 distributions. several channels can be fitted simultaneously.
12 
13 @section sec-mtf-math Mathematical formulation
14 
15 the multi-template fitter is formulated in terms of bayesian reasoning.
16 the posterior probability is proportional to the product of the
17 likelihood and the prior probability. the latter can be freely chosen by
18 the user whereas the likelihood is predefined. it is a binned likelihood
19 which assumes that the fluctuations in each bin are of poisson nature
20 and independent of each other. all channels, processes and sources of
21 systematic uncertainties are assumed to be uncorrelated.
22 
23 the parameters of the model are thus the expectation values of the
24 different processes, \f$\lambda_{k}\f$, and the nuisance parameters,
25 \f$\delta_{l}\f$.
26 
27 @subsection sec-mtf-exclsyst Excluding systematic uncertainies
28 
29 in case no sources of systematic uncertainty are taken into account the
30 likelihood is defined as
31 \f{eqnarray}{
32 l = \prod_{i=1}^{n_{\mathrm{ch}}} \prod_{j=1}^{n_{\mathrm{bin}}} \frac{\lambda_{ij}^{n_{ij}}}{n_{ij}!} e^{-\lambda_{ij}} \, ,
33 \label{eqn:likelihood}
34 \f}
35  where \f$n_{\mathrm{ch}}\f$ and
36 \f$n_{\mathrm{bin}}\f$ are the number of channels and bins, respectively.
37 \f$n_{ij}\f$ and \f$\lambda_{ij}\f$ are the observed and expected number of
38 events in the \f$j\f$th bin of the \f$i\f$th channel. The expected number of
39 events are calculated via
40 \f{eqnarray}{
41 \lambda_{ij} & = & \sum_{k=1}^{N_{\mathrm{p}}} \lambda_{ijk} \\
42  & = & \sum_{k=1}^{N_{\mathrm{p}}} \lambda_{k} \cdot f_{ijk} \cdot \epsilon_{ik} \, ,
43 \label{eqn:expectation}
44 \f}
45  where \f$f_{ij}\f$ is the bin content
46 of the \f$j\f$th bin in the normalized template of the \f$k\f$th process in the
47 \f$i\f$th channel. \f$\epsilon_{ik}\f$ is the efficiency of the \f$k\f$th process in
48 the \f$i\f$th channel specified when setting the template. \f$\lambda_{k}\f$ is
49 the contribution of the \f$k\f$th process and is a free parameter of the
50 fit.
51 
52 @subsection sec-mtf-inclsyst Including Systematic Uncertainies
53 
54 In case sources of systematic uncertainties are taken into account, the
55 efficiency \f$\epsilon_{ik}\f$ is modified according to a nuisance
56 parameter:
57 \f{eqnarray}{
58 \epsilon_{ik} \rightarrow \epsilon_{ik} \cdot (1 + \sum_{l=1}^{N_{\mathrm{syst}}} \delta_{l}\cdot \Delta\epsilon_{ijkl}) \, ,
59 \label{eqn:systematic}
60 \f}
61  where \f$\delta_{l}\f$ is the nuisance
62 parameter associated with the source of systematic uncertainty and
63 \f$\Delta\epsilon_{ijkl}\f$ is the change in efficiency due to the \f$l\f$th
64 source of systematic uncertainty in the \f$i\f$th channel and \f$j\f$th bin for
65 the \f$k\f$th process.

79 @section sec-mtf-addchannel Adding a Channel
80 
81 The MTF fits several channels simultaneously. These channels can be
82 physics channels, e.g., \f$Z^{0}\rightarrow e^{+}e^{-}\f$ and
83 \f$Z^{0}\rightarrow \mu^{+}\mu^{-}\f$, samples with disjunct jet
84 multiplicity or entirely different classes altogether. A new channel can
85 be added using the following method:
86 
87  @code{.cpp}
88  int BCMTF::AddChannel(const char * name)
89  @endcode
90 
91 where `name` is the name of the process, and the return value is an
92 error code. Note that at least one channel has to be added.
93 
94 @section sec-mtf-adddata Adding a Data Set
95 
96 Each channel added to the MTF has a unique data set which comes in form
97 of a (`TH1D`) histogram. It can be defined using the following method:
98 
99  @code{.cpp}
100  int BCMTF::SetData(const char * channelname, TH1D hist)
101  @endcode
102 
103 where `channelname` is the name of the channel and `hist` is the
104 histogram representing the data. The return value is an error code.

106 @section sec-mtf-addprocess Adding a Process
107 
108 Each template that is fit to the data set corresponds to a process,
109 where one process can occur in several channels. The fit then defines
110 the contribution of the process and thus each process comes with one
111 model parameter. A process can be added using the following method:
112 
113  @code{.cpp}
114  int BCMTF::AddProcess(const char * name,
115  double nmin = 0.,
116  double nmax = 1.) ,
117  @endcode
118 
119 where `name` is the name of the process and `nmin` and `nmax` are the
120 lower and upper bound of the parameter associated with the contribution
121 of the process. The parameter is denoted
122 \f$\lambda_{k} \, (k=1\dots N_{\mathrm{p}})\f$ in the @ref sec-mtf-math.
123 Note that at least one process has to be added. A prior needs to be
124 defined for each process, using the default `BCModel` methods.
125 
126 It is likely that a single process will have different shapes in
127 different channels. Thus, templates for a process need to be defined for
128 each channel separately using the following method:
129 
130  @code{.cpp}
131  int BCMTF::SetTemplate(const char * channelname,
132  const char * processname,
133  TH1D hist,
134  double efficiency = 1.) ,
135  @endcode
136 
137 where `channelname` and `processname` are the names of the channel and
138 the process, respectively. The parameter `hist` is the (`TH1D`)
139 histogram (or template) which represents the process. The histogram will
140 be normalized to unity and the entries in the normalized histogram are
141 the probabilities to find an event of a process \f$k\f$ and channel \f$i\f$ in
142 bin \f$j\f$. This probability is denoted
143 \f$f_{ijk}\,(i=1\dots N_{\mathrm{ch}}, \, j=1\dots N_{\mathrm{b}}, \,
144 k=1\dots N_{\mathrm{p}})\f$ in the @ref sec-mtf-math. The last
145 parameter, `efficiency`, is the efficiency of the process in that
146 channel and is used to scale to template during the fit. This is needed
147 if a process contributes with different amounts in two separate
148 channels. The efficiency is denoted \f$\epsilon_{ik} \, (i=1\dots
149 N_{\mathrm{ch}}, \, k=1\dots N_{\mathrm{p}})\f$ in
150 the @ref sec-mtf-math. The return value is an error code. Note that
151 templates do not have to be set if the process does not contribute to a
152 particular channel.
153 
154 @section sec-mtf-addunc Adding Systematic Uncertainties
155 
156 Systematic uncertainties can alter the shape of a template. Sources of
157 systematic uncertainty can be included in the fit using nuisance
158 parameters. This nuisance parameter is assumed to alter the original
159 template linearly, where values of -1, 0, and 1 correspond to the
160 “downwards” shifted, nominal and “upwards shifted” template,
161 respectively. The nuisance parameters are denoted \f$\delta_{l} \,
162 (l=1\dots N_{\mathrm{syst}})\f$ in the @ref sec-mtf-math. Shifted
163 refers to a change of one standard deviation. An example for a nuisance
164 parameter could be the jet energy scale (JES). With a nominal JES of 1
165 and and uncertainty of 5%, the scaled templates correspond to a JES of
166 0.95 and 1.05, respectively. A prior needs to be defined for each
167 nuisance parameter which is usually chosen to be a standard normal
168 distribution. A source of systematic uncertainty can be added using the
169 following method:
170 
171  @code{.cpp}
172  int BCMTF::AddSystematic(const char * name,
173  double min = -5.,
174  double max = 5.) ,
175  @endcode
176 
177 where `name` is the name of the source of systematic uncertainty and
178 `min` and `max` are the lower and upper bound of the nuisance parameter,
179 respectively. The return value is an error code.
180 
181 Since the different sources of systematic uncertainty have an individual
182 impact on each process and in each channel, these need to be specified.
183 Two method can be used to define the impact:
184 
185  @code{.cpp}
186  int BCMTF::SetSystematicVariation(const char * channelname,
187  const char * processname,
188  const char * systematicname,
189  TH1D hist_up,
190  TH1D hist_down) ,
191  @endcode
192 
193 where `channelname`, `processname` and `systematicname` are the names of
194 the channel, the process and the source of systematic uncertainty. The
195 (`TH1D`) histograms `hist_up` and `hist_down` are the histograms
196 corresponding to an “up”- and “down”-scaling of the systematic
197 uncertainty of one standard deviation, i.e., for each bin entry \f$y\f$ they
198 are calculated as
199 \f{eqnarray}{
200 \label{eqn:deltay}
201 \Delta_{\mathrm{up}} & = & (y_{\mathrm{up}} - y_{\mathrm{nominal}})/y_{\mathrm{nominal}} \, , \\
202 \Delta_{\mathrm{down}} & = & (y_{\mathrm{nominal}} - y_{\mathrm{down}})/y_{\mathrm{nominal}} \, .
203 \f}
204 
205 Note the sign of the down-ward fluctuation. These histograms define the
206 change of the bins in each template in the efficiency which is denoted
207 \f$\Delta\epsilon_{ijkl} \, (i=1\dots N_{\mathrm{ch}}, \,
208 j=1\dots N_{\mathrm{b}}, \, k=1\dots N_{\mathrm{p}}, \, l=1\dots
209 N_{\mathrm{syst}})\f$. For example, if the value for a particular bin of
210 `hist_up` is 0.05, i.e., if the systematic uncertaintiy is 5% in that
211 bin, then the efficiency of the process in that channel will be
212 multiplied by \f$(1+0.05)\f$. The return value is an error code.
213 
214 The second variant does not take the difference in efficiency, but
215 calculates it internally from the absolute values:
216 
217  @code{.cpp}
218  int BCMTF::SetSystematicVariation(const char * channelname,
219  const char * processname,
220  const char * systematicname,
221  TH1D hist,
222  TH1D hist_up,
223  TH1D hist_down) ,
224  @endcode
225 
226 where `channelname`, `processname` and `systematicname` are the names of
227 the channel, the process and the source of systematic uncertainty. The
228 (`TH1D`) histograms `hist`, `hist_up` and `hist_down` are the nominal
229 histogram and the histograms corresponding to an “up”- and
230 “down”-scaling of the systematic uncertainty of one standard deviation.
231 In this case, the histograms are not the relative differences but the
232 absolute values. The return value is an error code.

244 @section sec-mtf-mtfoutput Output
245 The MTF produces several outputs:
246 
247 1. `PrintAllMarginalized(const char* name)` prints the marginalized
248 distributions in 1D and 2D for all parameters, i.e., the processes
249 and nuisance parameters into a PostScript file `name`.
250 
251 2. `PrintResults(const char* name)` writes a summary of the fit into a
252 text file `name`.
253 
254 3. To print a stacked histogram of the templates and the data histogram in
255 the file `name` using a set of parameters `parameters`, do
256 @code{.cpp}
257 PrintStack(int channelindex,
258  const std::vector<double> & parameters,
259  const char * filename = "stack.pdf",
260  const char * options = "")
261 PrintStack(const char * channelname,
262  const std::vector<double> & parameters,
263  const char * filename = "stack.pdf",
264  const char * options = "")
265 @endcode
266 
267 For example, these could be the best fit results. Several options can
268 be specified:
269 
270 - `logx`: uses a log-scale for the x-axis.
271 
272 - `logy`: uses a log-scale for the y-axis.
273 
274 - `logx`: plot the x-axis on a log scale
275 
276 - `logy`: plot the y-axis on a log scale
277 
278 - `bw`: plot in black and white
279 
280 - `sum`: draw a line corresponding to the sum of all templates
281 
282 - `stack`: draw the templates as a stack
283 
284 - `e0`: do not draw error bars
285 
286 - `e1`: draw error bars corresponding to sqrt(n)
287 
288 - `b0`: draw an error band on the expectation corresponding to the
289  central 68% probability
290 
291 - `b1`: draw bands showing the probability to observe a certain
292  number of events given the expectation. The green (yellow, red)
293  bands correspond to the central 68% (95%, 99.8%) probability
294 
295 @section sec-mtf-mtfsettings Settings
296 Several settings can be changed which impact the fit.
297 
298 - `SetFlagEfficiencyConstraint` sets a flag if the overall efficiency
299  (calculated from the value given when setting a template and the
300  corresponding systematic uncertainties) is constrained to be
301  between 0 and 1 or not. The default value is `true`.

313 @section sec-mtf-perf_ensemble Performing Ensemble Tests
314 
315 Ensemble testing is done in two steps: first, ensembles are generated
316 according to the processes defined in the MTF. The ensembles are stored
317 in root files. In a second step, the ensembles are analyzed using the
318 MTF specified.
319 
320 @section sec-mtf-create_ensem Creating Ensembles
321 
322 Ensembles can be generated using several methods. A single ensemble can
323 be generated using the following method:
324 
325  @code{.cpp}
326  std::vector<TH1D> BCMTFAnalysisFacility::BuildEnsemble(const std::vector<double> & parameters)
327  @endcode
328 
329 where `parameters` is a set of parameters which corresponds to those in
330 the template fitter, i.e., the process contributions and nuisance
331 parameters. For most applications, the best fit parameters of the data
332 set at hand is used. The return value is a set of histograms
333 corresponding to a pseudo data set for the different channels.
334 
335 A similar method is used to generate multiple ensembles:
336 
337  @code{.cpp}
338  std::vector<TH1D> BCMTFAnalysisFacility::BuildEnsembles(const std::vector<double> & parameters, int nensembles)
339  @endcode
340 
341 where `nensembles` is the number of ensembles to be generated. The
342 return value is a pointer to a `TTree` object in which the ensembles are
343 stored. The entries in the tree are the parameters and the number of
344 entries in each bin of the data histograms.
345 
346 The third method is based on a tree where the tree contains a set of
347 parameters for each ensemble. This option is preferred if, e.g., the
348 ensembles should be varied accoring to the prior probabilities. The
349 method used to generate ensembles is
350 
351  @code{.cpp}
352  std::vector<TH1D> BCMTFAnalysisFacility::BuildEnsembles(TTree * tree, int nensembles)
353  @endcode
354 
355 where `tree` is the input tree. Note that the ensembles are randomized,
356 i.e., the first event in the tree does not correspond to the first
357 ensemble. This is done to avoid biases if the tree itself is the output
358 of a Markov Chain.
359 
360 @section sec-mtf-analyze_ensem Analyzing Ensembles
361 
362 Ensemble tests can be performed usign the ensembles defined earlier or
363 using a set of parameters. In the former case, the method is:
364 
365  @code{.cpp}
366  TTree * BCMTFAnalysisFacility::PerformEnsembleTest(TTree * tree, int nensembles)
367  @endcode
368 
369 where `tree` is the tree of ensembles and `nensembles` is the number of
370 ensembles to be analyzed. The return value is a tree containing the
371 information about the analyzed ensemble. The list of variables is
372 
373 - `parameter_i`: the \f$i\f$th parameter value used at the generation of
374  the ensemble.
375 
376 - `mode_global_i`: the \f$i\f$th global mode.
377 
378 - `std_global_i`: the \f$i\f$th standard deviation evaluated with the
379  global mode.
380 
381 - `chi2_generated_i`: the \f$\chi^{2}\f$ calculated using the parameters
382  at generation of the ensemble for channel \f$i\f$.
383 
384 - `chi2_mode_i`: the \f$\chi^{2}\f$ calculated using the global mode
385  parameters for channel \f$i\f$.
386 
387 - `cash_generated_i`: the Cash statistic (Likelihood ratio) calculated
388  using the parameters at generation of the ensemble for channel \f$i\f$.
389 
390 - `cash_mode_i`: the Cash statistic (Likelihood ratio) calculated
391  using the global mode parameters for channel \f$i\f$.
392 
393 - `n_events_i`: the number of events in the ensemble in channel \f$i\f$.
394 
395 - `chi2_generated_total`: the total \f$\chi^{2}\f$ calculated using the
396  parameters at generation of the ensemble.
397 
398 - `chi2_mode_total`: the total \f$\chi^{2}\f$ calculated using the global
399  mode parameters.
400 
401 - `cash_generated_total`: the total Cash statistic calculated using
402  the parameters at generation of the ensemble.
403 
404 - `cash_mode_total`: the total Cash statistic calculated using the
405  global mode parameters.
406 
407 - `n_events_total`: the total number of events in the ensemble.
408 
409 Ensemble tests can also be performed using the following methd:
410 
411  @code{.cpp}
412  TTree * BCMTFAnalysisFacility::PerformEnsembleTest(const std::vector<double> & parameters, int nensembles)
413  @endcode
414 
415 in which case the ensembles are generated internally using the
416 parameters and are then analyzed.
417 
418 By default the log messages for both the screen and the log-file are
419 suppressed while performing the ensemble test. This can be changed using
420 
421  @code{.cpp}
422  void BCMTFAnalysisFacility::SetLogLevel(BCLog::LogLevel level)
423  @endcode

457 @subsection sec-mtf-perf_calib Performing Calibration Analyses
458 
459 Ensemble tests for different sets of parameters can be automized by
460 using the method
461 
462  @code{.cpp}
463  int BCMTFAnalysisFacility::PerformCalibrationAnalysis(const char * dirname,
464  const std::vector<double> & default_parameters,
465  int index,
466  const std::vector<double> & parametervalues,
467  int nensembles = 1000)
468  @endcode
469 
470 which can be used to easily generate calibration curves. The ensembles
